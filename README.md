# Система автоматической проверки ТЗ
## Elevator Pitch
Компании теряют недели разработки из-за некачественных технических заданий: отсутствуют требования, противоречивые формулировки, не определены критерии приёмки. Сам процесс согласования ТЗ воспринимается как рутина.

Система автоматически анализирует ТЗ и выявляет потенциальные проблемы за несколько минут. В отличие от ручной проверки, система за короткое время находит неясности, пропущенные разделы и риски изменений объёма работ. В результате команды уменьшают количество правок, избегают конфликтов с заказчиком и точнее оценивают сроки и бюджет.

## Функциональные требования и нефункциональные требования
### Получить указания на проблемы в ТЗ
Функциональные требования
- Система дает пользователю загрузить ТЗ
- Система анализирует ТЗ и выявляет проблемные части
- Система отображает список проблем
- Система подсвечивает проблемные места в тексте
- Система показывает описание каждой проблемы

Нефункциональные требования
- Загрузка ТЗ в популярных форматах (PDF, DOCX...)
- Анализ документа должен занимать небольшое время (не более 5 секунд на 10 страниц документа)
- Система не блокирует интерфейс во время анализа
- Система отображает progress bar во время анализа
- Система поддерживает английский и русский языки
- Система корректно отображается на экранах разного расширения
- Понятный интерфейс

### Оплата лицензии / подписки
Функциональные требования
- При нажатии на кнопку оплаты система запрашивает подтверждение
- Система изменяет статус заказчика при удачной оплате
- Система выдает предупреждение об ошибке в случае неудачной оплаты
- Система предлагает выбрать варианты подписки / лицензии (месяц, квартал, год)
- Система дает возможность отменить продление подписки
- Система показывает пользовательское соглашение
- Система поддерживает восстановление покупки

Нефункциональные требования
- Платеж должен обарабатываться не дольше 5 секунд
- Платежи защищены и безопасны
- поддержка нескольких платежных методов
- Понятный интерфейс

## BDD
### Критический путь
- Пользователь открывает приложение
- Пользователь загружает ТЗ
- Пользователь получает указание на проблемы
### ✅ Успешный сценарий
#### Дано
- Пользователь открыл приложение
- У пользователя есть файл ТЗ в поддерживаемом формате (PDF, DOCS...)
#### Когда
- Пользователь выбирает нужный файл в проводнике
- Пользователь нажимает "Загрузить ТЗ"
#### Тогда
- Закрывается окно проводника
- Система создает рабочую копию файла
- Система запускает анализ ТЗ
- Система отображает progress bar
- Система показывает указание на проблемы
### ❌ Неуспешный сценарий
#### Дано
- Пользователь открыл приложение
- У пользователя есть файл ТЗ в неподдерживаемом формате
#### Когда
- Пользователь выбирает нужный файл в проводнике
- Пользователь нажимает "Загрузить ТЗ"
#### Тогда
- Закрывается окно проводника
- Система отображает сообщение об ошибке и список поддерживаемых форматов
