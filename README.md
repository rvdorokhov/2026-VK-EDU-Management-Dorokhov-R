# Система автоматической проверки ТЗ
## Elevator Pitch
Компании теряют недели разработки из-за некачественных технических заданий: отсутствуют требования, противоречивые формулировки, не определены критерии приёмки. Сам процесс согласования ТЗ воспринимается как рутина.

Система автоматически анализирует ТЗ и выявляет потенциальные проблемы за несколько минут. В отличие от ручной проверки, система за короткое время находит неясности, пропущенные разделы и риски изменений объёма работ. В результате команды уменьшают количество правок, избегают конфликтов с заказчиком и точнее оценивают сроки и бюджет.

## Роли и User Stories

1. Автор / Проверяющий ТЗ 
- Загрузить ТЗ в систему - ✅ Must
- Получить указания на проблемы - ✅ Must
- Получить общую оценку качества - 🟡 Could
- Получить оценку критичности проблем - 🟢 Should
- Получить рекомендации по исправлению проблем - 🟢 Should
- Исправить проблемы вручную не выходя из системы - 🟢 Should
- Исправить проблемы автоматически не выходя из системы - ❌ Won’t
- Интеграция с Jira, Helix RM и т.п. - ❌ Won’t
- Использовать готовые шаблоны ТЗ - 🟡 Could
- Сравнить ТЗ до/после - 🟡 Could
- Посмотреть историю проверок - 🟡 Could
- Отметить проблему как "исправлено" - 🟡 Could
- Связаться с поддержкой - ✅ Must
- Оставить обратную связь - 🟡 Could

2. Руководитель
- Оплата лицензии / подписки - ✅ Must
- Отмена подписки - ✅ Must
- Предоставить подчиненным доступ к системе - ✅ Must
- Забрать у подчиненных доступ к системе - ✅ Must
- Развернуть систему на собственных серверах компании - 🟢 Should
- Собрать статистику - 🟡 Could
- Связаться с поддержкой - ✅ Must
- Оставить обратную связь - 🟡 Could

3. Поддержка
- Связаться с пользователем - ✅ Must
- Разрешить проблемы с подпиской / лицензией - ✅ Must
